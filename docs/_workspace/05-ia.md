## 2. Comparison Table — Columns, Rows, Unknown-Cell Treatment

**Columns (5):** `Feature` · **`VoiceToText`** · `Wispr Flow` · `Superwhisper` · `MacWhisper` · `Apple Dictation` — we keep the four competitors the README already name-checks. Apple Dictation is the free baseline; the other three are the paid category leaders. Aqua, Willow, BetterDictation are set aside for a future `/compare` deep-dive.

> **Why these 4 (and not Willow / Aqua / BetterDictation):** T1 confirms Wispr/Super/MacWhisper/Apple are the most searched and most recognized. T2 §1.1 and §5 repeatedly anchors SEO on the same four. Matching the README keeps messaging consistent. Willow/Aqua are cloud-first and lesser-known; BetterDictation has no free tier — weaker comparison frames.

**Rows (10)** — drawn from T1 §"Proposed Feature Comparison Table" and tuned with T4's opinionated, label-not-checkmark copy rule. Left-aligned feature names in `body` 500; cell values in `body-s` 400 `text-secondary`; **descriptive labels, never bare ✓/✗**.

| # | Feature (row) | VoiceToText | Wispr Flow | Superwhisper | MacWhisper | Apple Dictation |
|---|---|---|---|---|---|---|
| 1 | Price | Free forever | $12–15 / mo | $8.49 / mo · paid lifetime | $69–80 lifetime (or sub.) | Free (built-in) |
| 2 | Open source | Yes — MIT on GitHub | No | No | No | No |
| 3 | Runs 100% offline | On-device | Cloud only | On-device | On-device | On-device (Apple Silicon, some langs) |
| 4 | No account / no login | None required | Account required | Account required | No account | None required |
| 5 | Push-to-talk (hold key) | Hold `⌥ Space` | Customizable hold | Customizable hold | Pro (Global mode) | Toggle only |

| 7 | Apple Neural Engine acceleration | Native | Not advertised | Yes | Yes | Yes |
| 8 | Native macOS app (no Electron) | SwiftUI native | Cross-platform shell | Native | Native | Native |

| 10 | Tuned for AI coding agents (Claude Code / Cursor / Codex) | Yes — headline use case | Generic "works in apps" | Yes — messaged | Generic "works in apps" | No |

### Visual treatment (for T7/T8, per T4 §4)

- Outer card: `bg-1` (#121214), 16px radius, 1px `border-subtle`.
- 5-col grid; VoiceToText column: `bg` slightly elevated (`#17171A`), 2px cyan inset-top border (`rgba(0,212,255,0.4)`), column header gets mono-accent wordmark.
- Row separator: `border-subtle` hairline, no heavy borders. Row hover: `bg-2` (#17171A).
- Status glyph palette (use next to or replacing the text label where a glyph adds clarity):
  - **Full / good-for-us:** `#3DDC97` filled dot + label ("Free forever", "On-device", "Yes").
  - **Partial:** `#F5A524` half-dot + label ("Pro only", "Trial only", "Cloud-assisted").
  - **Missing / weaker:** `#71717A` em-dash + label ("Cloud only", "Toggle only", "—").
- **"Unknown" cell treatment (honesty rule):** where T1 marked a competitor fact as **unknown / not advertised**, we **do not guess**. Render the cell as:

  > `text-tertiary` em-dash `—` + small-caps footnote marker (`ⁿ`), label "Not advertised." Footnote block directly below the table lists what was unverifiable and links to the source URL. Example cell for "Apple Neural Engine acceleration / Wispr Flow": `— Not advertised¹`.

  Footnotes list (shown as a muted caption under the table):
  - `¹ Wispr Flow does not advertise Neural Engine acceleration; treat as unverified.`
  - `² Superwhisper lifetime pricing was in flux at research time — re-verify before each release.`
  - `³ MacWhisper Pro price is reported as $69 / $79.99 / €59 across sources — re-verify on the Gumroad page.`

- Mobile (≤ 768px): collapse to 4 stacked cards, **VoiceToText pinned first**. Each card shows the 10 rows as label/value pairs.

**Do NOT** put competitor logos in the header. Text wordmarks only (T4 §4, T3 §5). Tone stays neutral — the table does the work (T1 §"Copy + IA Recommendations").

## 3. CTA Hierarchy

| Rank | CTA label | Href | Style | Appears in | Notes |
|---|---|---|---|---|---|
| Primary | **Download for Mac** | `https://github.com/gug007/voice-to-text/releases/latest/download/VoiceToText.dmg` | Gradient (`#00D4FF → #7B8CFF`), 48px tall, 12px radius — used only here, nowhere else. | Hero (right-side), AI-agent section (repeat), Download section (large, centered). **3 placements total.** | Fires a GA-free event if analytics added later; for now, just a plain `<a download>`. |
| Secondary | **View on GitHub** | `https://github.com/gug007/voice-to-text` | Ghost button — 1px `border-subtle`, transparent bg, `text-primary`. | Hero (next to primary), Top nav (right side), Footer. | Opens in new tab. `rel="noopener"`. |
| Tertiary (contextual, not buttons) | "Read the source" / "Read the code" / "See the license" | Repo / `LICENSE` / `README.md` anchor | Inline text link, `accent` underline-on-hover. | Social proof strip, Features (privacy block), Comparison footnotes, FAQ, Footer. | Invites audit; satisfies T3 Segment C objection. |

### Rules
- **Primary CTA count** on any single screen: max **1** visible at a time. Hero pairs Primary + Secondary; AI-agent and Download sections each have a single gradient Primary.
- **Nav CTA:** only the secondary "View on GitHub" lives in the sticky nav. Primary Download is reachable via `#download` anchor — avoid double-CTAs in nav (T4 anti-pattern "overloaded nav").
- **No waitlist, no email capture, no signup.** This is free OSS (T4 anti-pattern #7).

## 4. Above-the-Fold Budget (1440 × 900)

Assumption: `1440×900` with a ~70–100px browser chrome + ~80px sticky nav leaves ~**720–750px** of usable hero viewport. Everything below must fit, with the requirements chip **inside** the fold.

| Element | Pixel budget | Rationale |
|---|---|---|
| Sticky top nav | ~64px h | Wordmark `VoiceToText` (left) · `Features · Compare · FAQ` center · **View on GitHub** (right). Max 5 items. |
| Hero vertical padding (top) | ~80–100px | Airy, T4 §3 rhythm |
| H1 headline (`display-xl`, 2 lines) | ~180–220px | "Dictate into any app." / "On-device. Free forever." — or equivalent T6 final copy. Max 10 words. |
| Keycap SVG + waveform | ~180–200px | `⌥` + `Space` keycap pair centered; 5–7 CSS waveform bars under them |
| Sub-headline (`body-l`, 3 lines, ≤560px wide) | ~90–100px | Names apps (Claude Code, Cursor, Slack), offline, ANE — see T3 §3 sub-headline |
| CTA row (Primary + Secondary) | ~56–64px h | 48px buttons + 12–16px gap |
| Requirements chip (`Free · macOS 14+ · Apple Silicon`) | ~24–28px | Mono, `text-tertiary` — **must be in fold** to preempt T3 objection "I'm on Intel Mac" |
| **Total** | **≈ 700–760px** | Fits within 720–750px usable viewport on 1440×900 with minimal slack. |

### Must be visible without scrolling on 1440×900
- VoiceToText wordmark + nav
- H1 headline
- Keycap visual (signals "press this")
- One paragraph of sub-copy with at least one named app
- **Primary Download CTA**
- **Secondary GitHub CTA**
- Requirements chip

### Explicitly **NOT** in the fold
- The demo.gif (it's in its own section to preserve LCP — T4 §5; T2 H7)
- Social proof strip (sits right below the hero; visible after ~100px scroll)
- Any comparison or FAQ content

### 1280×800 (MacBook Air 13" baseline)
Drop one line of sub-copy and tighten vertical padding to 60/60. Everything above is still in-fold.

### 375×812 (mobile)
Fold shows: nav (hamburger) + H1 (reflowed) + keycap (scaled to ~96px) + one CTA only (**Download for Mac**); secondary GitHub moves to a text link below. Sub-copy truncated to 2 lines.

## 7. Quick Reference — One-Screen Summary

**Final section order:** Hero → Social-proof strip → Demo showcase → How-it-works (3 steps) → Features → AI-agent use case → Comparison table → FAQ → Download/Install → Footer.

**Comparison table shape:** 5 cols × 10 rows. Cols: Feature, **VoiceToText**, Wispr Flow, Superwhisper, MacWhisper, Apple Dictation. "Unknown" cells render as `— Not advertised` with a numbered footnote.

**Above fold (1440×900):** nav · H1 · keycap + waveform SVG · sub-headline · both CTAs · requirements chip.

**Blocking missing assets:** OG image, favicon set, keycap + waveform SVGs, JSON-LD blocks, robots.txt/sitemap.xml, shields.io badges, verified license file.
