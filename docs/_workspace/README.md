# VoiceToText landing — team workspace

Research briefings, specs, and audit reports produced by the landing-page
build team. Kept in-repo for future contributors who want to understand
the reasoning behind copy, IA, visual, a11y, perf, and SEO choices.

Not served by GitHub Pages as a user-facing page (no `index.html` here).

- `01-competitors.md` — competitor & positioning research (T1)
- `02-seo-plan.md` — keyword strategy, meta plan (T2)
- `03-audience.md` — audience segments & FAQ seeds (T3)
- `04-design-trends.md` — 2026 landing-page design benchmarks (T4)
- `05-ia.md` — information architecture + section order (T5)
- `06-copy.md` — final body copy, headlines, microcopy (T6)
- `07-design-system.md` — tokens, components, visual spec (T7)
- `10-perf-report.md` — Core Web Vitals pass + critical CSS log (T10)
- `11-a11y-report.md` — WCAG 2.2 AA audit + fixes (T11)
- `12-qa-report.md` — final QA checklist + ship verdict (T12)
